# 2026-05 第 3 周执行清单

## 一周目标

- 读懂 logs、topics、event 的基本机制，让 CLI 工具能看懂回执和事件。

## 必读资料

- [Transactions](https://ethereum.org/developers/docs/transactions/)
- [Ethereum JSON-RPC API](https://ethereum.org/en/developers/docs/apis/json-rpc/)
- [Geth JSON-RPC Server](https://geth.ethereum.org/docs/interacting-with-geth/rpc)

## 本周必须完成的小 Demo

- 场景：把工具从“只能查交易”升级到“能辅助分析交易执行结果”，更接近真实排查问题的需求。
- 功能要求：
  - 输入交易哈希后，增加查询 receipt 的能力。
  - 输出 `status`、`gasUsed`、`blockNumber`、`logs` 数量。
  - 如果存在 logs，至少展示每条 log 的 address、topic0、data 长度。
  - 输出格式尽量清晰，方便复制查看。
- 完成效果：
  - 你能用自己的工具判断一笔交易是否成功、消耗了多少 gas、有没有事件日志。
  - 工具开始具备“交易排查助手”的味道。
  - README 中增加一段 receipt 查询示例。
